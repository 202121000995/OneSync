# OneSync V1 架构审核

## 审核范围

- `architecture.md`
- OneSync V1 MVP 边界
- 模块职责与依赖方向
- 核心接口与数据结构
- 网络协议、安全要求和测试策略

## 审核结论

通过，可以进入 `feature/project-bootstrap`。

## 审核结果

### 架构一致性

- 模块划分符合项目总规约。
- 同步核心、网络层、Relay 和 Web 管理边界清晰。
- V1 保持单向同步，没有引入双向同步、多用户、集群等范围外功能。
- 开发任务顺序与 Phase 1 至 Phase 8 基本一致。

### 代码质量

- 当前阶段没有业务代码。
- 核心接口使用 Go 标准库类型，未要求额外依赖。
- 数据结构足以支撑扫描、差异、传输和任务编排的后续设计。

### 安全

- 已覆盖路径穿越、绝对路径注入、消息长度限制和令牌日志脱敏。
- 已明确临时文件校验后原子替换。
- 已明确禁止危险删除根目录、空路径和 `.`。
- TCP 是否启用 TLS 仍是进入网络开发前的阻塞确认项。

### 性能

- 文件采用分块传输，避免整文件进入内存。
- 哈希可按需计算，避免扫描阶段无条件读取全部内容。
- 单任务串行同步周期可降低并发写冲突风险。
- 并发传输策略可以推迟到正确性验证完成后决定。

## 后续阻塞项

以下问题不阻塞项目骨架、文件扫描和文件差异阶段，但必须在对应模块开发前确认：

- TCP 与 Relay 的 TLS 要求。
- 同步链接有效期和重复使用规则。
- 目标端额外文件的自动删除策略。
- 文件冲突时的覆盖规则。
- 符号链接策略。

## 文件扫描审核

审核分支：`feature/file-scanner`

审核结论：通过。

审核说明：

- 实现与架构中的 `Scanner`、`Snapshot` 和 `FileEntry` 契约一致。
- 仅扫描普通文件，忽略符号链接、命名管道和其他特殊文件。
- 根目录为空、缺失、不是目录或本身为符号链接时返回错误。
- 快照路径统一使用 `/` 分隔，并保持相对同步根目录。

- 扫描支持上下文取消，扫描期间条目消失时安全跳过。
- 未引入第三方依赖，未修改其他模块。

审核中修复：

- 根目录本身是符号链接时曾可能返回空快照，现已改为明确拒绝。

## 文件差异审核

审核分支：`feature/file-differ`

审核结论：通过。

已确认产品规则：

- 目标端独有文件保留，不自动删除。
- 同路径文件冲突时以源端为准，生成覆盖操作。

审核说明：

- 仅源端存在的文件生成 `create`。
- 同路径内容不同的文件生成 `update`，操作携带源端条目。
- 双方都有哈希时比较哈希，否则比较大小和修改时间。
- 权限模式不参与内容判断，避免 Windows 与 Linux 间反复更新。
- 操作按规范化相对路径稳定排序。
- 拒绝绝对路径、路径穿越、反斜杠、盘符/冒号、空字节及键值不一致的快照。
- 未引入第三方依赖，未接入传输或删除逻辑。

审核中修复：

- 移除权限模式差异触发更新的逻辑，避免跨平台循环同步。
- 增加跨平台危险路径校验。

验证结果：

- `go test ./...` 通过。
- `go vet ./...` 通过。
- `go test -race ./internal/sync` 通过。
- Windows amd64 测试编译通过。
- Linux amd64 测试编译通过。

剩余风险：

- 未计算哈希时，大小与修改时间相同但内容不同的文件无法在差异阶段识别；正式同步前应按策略补算哈希。

## TLS TCP 协议审核

审核分支：`feature/tcp-protocol`

审核结论：通过。

已确认产品规则：

- V1 的 TCP 与 Relay 通信强制使用 TLS。
- 客户端必须验证服务端证书，不允许跳过验证。

审核说明：

- 协议使用固定长度头和大端序字段，包含版本、类型、请求 ID 和载荷长度。
- 单帧载荷默认及绝对上限为 16 MiB，可配置更小限制。
- 解码在分配载荷内存前检查协议版本、消息类型和长度。
- 会话支持并发一个发送与一个接收，并支持上下文取消和截止时间。
- TLS 客户端与服务端最低版本固定为 TLS 1.3。
- TLS 证书、私钥和信任根由调用方配置，网络层不生成或持久化密钥。
- 同步令牌至少 32 字节，服务端使用固定长度 SHA-256 摘要进行恒定时间比较。
- 认证失败只返回通用错误，不泄露令牌差异。
- 未引入第三方依赖，尚未接入程序入口。

审核中修复：

- 监听取消产生的底层超时统一映射为上下文错误。
- 取消监视协程退出后才重置连接截止时间，避免污染后续操作。
- TLS 接入串行化，避免并发修改监听截止时间。
- 清理架构中与“不删除目标端独有文件”冲突的旧表述。

验证结果：

- `go test ./...` 通过。
- `go vet ./...` 通过。
- `go test -race -count=5 ./internal/network` 通过。
- TLS 证书验证与真实回环连接集成测试通过。
- Windows amd64 测试编译通过。
- Linux amd64 测试编译通过。

剩余风险：

- 证书签发、轮换和配置文件存储尚未实现，属于后续配置与任务管理范围。
- 当前帧层允许 16 MiB，但文件传输阶段仍应使用更小的固定块尺寸。

## 分块文件传输审核

审核分支：`feature/file-transfer`

审核结论：通过。

平台决策：

- 当前 Go 主客户端支持 Windows 10 及以上。
- Windows 7 后续作为独立兼容客户端评估，不限制使用 Go。

审核说明：

- 文件块最大为 256 KiB，发送与接收内存占用有界。
- 接收端每块写入并刷新后确认偏移，断线重连可从已确认长度继续。
- 文件 ID 由规范路径和 SHA-256 推导，拒绝对端伪造的恢复标识。
- 完成时重新校验文件大小和 SHA-256，失败不会覆盖正式文件。
- 损坏的临时文件在最终哈希失败后清除，下一次从零开始。
- 临时目录权限为 `0700`，临时文件权限为 `0600`。
- 拒绝绝对路径、路径穿越、盘符、反斜杠、空字节和符号链接父目录。
- 接收根目录和临时目录不能是符号链接。
- 接收前检查目标卷剩余空间，仅要求尚未续传部分所需容量。
- Unix 使用原子重命名；Windows 使用 `MoveFileExW` 替换并写透。
- 未引入第三方依赖，尚未接入同步引擎或程序入口。

审核中修复：

- 增加接收根目录、父目录和临时目录的符号链接防护。
- 验证文件 ID 与路径、哈希一致，避免临时文件标识碰撞。
- 最终哈希失败时清除损坏临时文件。
- 增加 Unix/Windows 可用磁盘空间检查。

验证结果：

- `go test ./...` 通过。
- `go vet ./...` 通过。
- `go test -race ./internal/transfer` 通过。
- 多块大文件、覆盖、空文件和真实断线重连续传测试通过。
- Windows amd64 测试编译通过。
- Linux amd64 测试编译通过。

剩余风险：

- 同一目标任务的并发传输必须由后续同步引擎串行化，避免多个会话同时操作同一临时文件。
- 对本机高权限进程在路径检查后立即替换目录项的竞争攻击，仍需操作系统级目录句柄方案才能完全消除。

## 同步引擎审核

审核分支：`feature/sync-engine`

审核结论：通过。

审核说明：

- 源端请求目标快照、扫描源目录、计算差异并按稳定顺序发送文件。
- 目标端生成快照、确认操作数量、逐文件接收并确认同步完成。
- 同一任务的并发触发合并为一次运行，等待者共享运行结果。
- 一个引擎实例绑定一个任务和会话，拒绝不同任务复用导致消息交错。
- 目标端独有文件不产生操作，端到端测试确认其保持不变。
- 源端和目标端默认扫描均计算 SHA-256，避免仅依赖大小和修改时间。
- 远端快照使用标准 JSON 编码，拒绝未知字段、危险路径和超量文件。
- 单轮操作数量上限为 1,000,000。
- `.onesync-part` 被定义为保留目录，扫描器整目录忽略。
- 源文件使用同一打开句柄完成哈希和发送，并拒绝符号链接替换。
- 未引入第三方依赖，尚未接入任务持久化或程序入口。

审核中修复：

- 排除 `.onesync-part`，避免未完成文件进入下一轮快照。
- 修正同任务合并测试时序，重复竞态测试稳定通过。
- 引擎实例绑定任务 ID，避免不同任务在同一会话交错。
- 严格拒绝快照 JSON 未知字段。
- 源文件发送前核对路径条目和打开文件身份，拒绝符号链接。

验证结果：

- `go test ./...` 通过。
- `go vet ./...` 通过。
- `go test -race -count=5 ./internal/sync ./internal/transfer` 通过。
- 新增、覆盖、目标端独有文件保留和零差异端到端测试通过。
- Windows amd64 同步引擎测试编译通过。
- Linux amd64 同步引擎测试编译通过。

剩余风险：

- 网络断开后的自动重新拨号与重新认证尚未由任务管理器编排；文件级续传能力已经具备。
- 任务状态、进度和最近错误尚未持久化，属于下一阶段任务管理范围。
