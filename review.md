## 审核结果

### 架构一致性

- 模块划分符合项目总规约。

- 已覆盖路径穿越、绝对路径注入、消息长度限制和令牌日志脱敏。

## TLS TCP 协议审核

审核分支：`feature/tcp-protocol`

审核结论：通过。

已确认产品规则：

- V1 的 TCP 与 Relay 通信强制使用 TLS。
- 客户端必须验证服务端证书，不允许跳过验证。

审核说明：

- 协议使用固定长度头和大端序字段，包含版本、类型、请求 ID 和载荷长度。
- 单帧载荷默认及绝对上限为 16 MiB，可配置更小限制。
- 解码在分配载荷内存前检查协议版本、消息类型和长度。
- 会话支持并发一个发送与一个接收，并支持上下文取消和截止时间。
- TLS 客户端与服务端最低版本固定为 TLS 1.3。
- TLS 证书、私钥和信任根由调用方配置，网络层不生成或持久化密钥。
- 同步令牌至少 32 字节，服务端使用固定长度 SHA-256 摘要进行恒定时间比较。
- 认证失败只返回通用错误，不泄露令牌差异。
- 未引入第三方依赖，尚未接入程序入口。

审核中修复：

- 监听取消产生的底层超时统一映射为上下文错误。
- 取消监视协程退出后才重置连接截止时间，避免污染后续操作。
- TLS 接入串行化，避免并发修改监听截止时间。
- 清理架构中与“不删除目标端独有文件”冲突的旧表述。

验证结果：

## 分块文件传输审核

审核分支：`feature/file-transfer`

审核结论：通过。

平台决策：

- 当前 Go 主客户端支持 Windows 10 及以上。
- Windows 7 后续作为独立兼容客户端评估，不限制使用 Go。

审核说明：

- 文件块最大为 256 KiB，发送与接收内存占用有界。
- 接收端每块写入并刷新后确认偏移，断线重连可从已确认长度继续。
- 文件 ID 由规范路径和 SHA-256 推导，拒绝对端伪造的恢复标识。
- 完成时重新校验文件大小和 SHA-256，失败不会覆盖正式文件。
- 损坏的临时文件在最终哈希失败后清除，下一次从零开始。
- 临时目录权限为 `0700`，临时文件权限为 `0600`。
- 拒绝绝对路径、路径穿越、盘符、反斜杠、空字节和符号链接父目录。
- 接收根目录和临时目录不能是符号链接。
- 接收前检查目标卷剩余空间，仅要求尚未续传部分所需容量。
- Unix 使用原子重命名；Windows 使用 `MoveFileExW` 替换并写透。
- 未引入第三方依赖，尚未接入同步引擎或程序入口。

审核中修复：

- 增加接收根目录、父目录和临时目录的符号链接防护。
- 验证文件 ID 与路径、哈希一致，避免临时文件标识碰撞。
- 最终哈希失败时清除损坏临时文件。
- 增加 Unix/Windows 可用磁盘空间检查。

验证结果：

- `go test ./...` 通过。
- `go vet ./...` 通过。
- `go test -race ./internal/transfer` 通过。
- 多块大文件、覆盖、空文件和真实断线重连续传测试通过。
- Windows amd64 测试编译通过。
- Linux amd64 测试编译通过。

剩余风险：

- 同一目标任务的并发传输必须由后续同步引擎串行化，避免多个会话同时操作同一临时文件。

## 任务管理审核

审核分支：`feature/task-manager`

审核结论：通过。

审核说明：

- 支持创建、启动、停止、读取和按 ID 排序列出任务。
- 状态包含 `created`、`connecting`、`syncing`、`idle`、`failed` 和 `stopped`。
- 启动异步执行，运行器创建阶段为 `connecting`，同步执行阶段为 `syncing`。
- 用户停止通过上下文取消并等待最终状态持久化。
- 成功进入 `idle`，运行或连接失败进入 `failed` 并保存最近错误。
- 进程重启时将遗留的 `connecting/syncing` 恢复为 `failed`。
- 任务文件采用版本 1 严格 JSON，拒绝未知字段、重复 ID 和非法状态。
- 状态文件最大 64 MiB，任务数量最大 10,000。
- 同步根目录必须使用绝对路径，避免重启工作目录变化。
- 原子写入使用同目录临时文件、刷新、关闭和替换。
- Unix 刷新父目录；Windows 使用 `MoveFileExW` 替换并写透。
- 状态文件权限为 `0600`，新建存储目录权限为 `0700`。
- 连接、上下文、运行器和瞬时进度不写入持久化文件。
- 状态文件路径由调用方提供，平台默认目录推迟到客户端集成阶段。
- 未引入第三方依赖，尚未接入真实网络运行器工厂或程序入口。

审核中修复：

- 最终状态持久化失败会返回给停止调用，并在内存中标记任务失败。
- 拒绝从符号链接或非普通文件读取任务状态。
- 持久任务必须有合法状态和有效时间戳。
- 限制状态文件、地址字段、根路径和错误文本大小。
- 拒绝运行器工厂返回空运行器。

## Relay 审核

审核分支：`feature/relay`

审核结论：通过。

审核说明：

- Relay 使用独立 TLS 1.3 服务端入口，启动时必须提供证书和私钥。
- 登记协议包含版本、源端或目标端角色、会话 ID 和 32 字节令牌。
- 仅会话 ID 相同、角色互补且令牌一致的连接可以配对。
- 令牌进入 Relay 后立即转换为 SHA-256 摘要，并使用恒定时间比较。
- Relay 不解析同步协议、不保存文件，只透明转发双向字节流。
- 会话日志仅记录会话 ID 的截断摘要，不记录令牌、本地路径或文件内容。
- 等待配对、活动会话和总连接数均有硬上限。
- 未发送完整登记信息的慢连接受登记超时和总连接数限制。
- 已配对连接受空闲超时和每方向字节上限限制。
- 上下文取消会打断登记、等待配对和活动转发，服务关闭时等待连接可及时退出。
- Relay 客户端登记接口支持源端和目标端角色，并等待双方配对确认。
- 未引入第三方依赖。

审核中修复：

- 配对后仅由等待方拥有转发循环，避免双方重复启动双向复制。

## 源端证书地址自动检测审核

审核分支：`feature/auto-detect-source-hosts`

审核结论：通过。

审核说明：

- Windows 验收包新增 `detect-source-hosts.ps1`，从 `ipconfig` 输出中提取私有 IPv4 地址，避免用户手工编辑脚本里的示例 IP。
- Windows `make-source-cert.cmd` 自动调用检测脚本，并把检测到的地址、`localhost` 和 `127.0.0.1` 写入源端证书。
- Linux `make-source-cert.sh` 自动从 `ip`、`hostname -I` 或 `ifconfig` 输出中提取私有 IPv4 地址。
- 用户仍可通过 `SOURCE_HOSTS` 覆盖自动检测结果，适配多网卡、VPN 或特殊网络。
- 打包脚本会把 Windows `.ps1` 辅助脚本放入验收包。
- Quickstart 和 preflight checklist 已改为“网页选择/手动输入可达地址”，不再要求先修改脚本里的内网 IP。

验证结果：

- `for script in packaging/acceptance-scripts/linux/*.sh; do sh -n "$script" || exit 1; done` 通过。
- `GO_BIN=/Users/apple/Library/Go/sdk/go1.26.3/bin/go packaging/package-acceptance.sh /private/tmp/onesync-acceptance-artifacts-hosts /private/tmp/onesync-acceptance-packages-hosts` 通过。
- `go test ./...` 通过。
- `go vet ./...` 通过。

剩余风险：

- 自动检测只选择私有 IPv4 地址，不会自动判断哪个地址对目标端实际可达；网页仍保留候选选择和手动输入。
- 如果源端 IP 后续变化，需要重新运行证书脚本并重新生成同步链接。
